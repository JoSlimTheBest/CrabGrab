Все фрагменты спрайта расположены так, чтобы нарезать их кратно размеру тайла 16х16!
То есть для легкого и правильного объединения фрагментов нужно захватывать прямоугольными выделениями кратно 16 пикселям!

seascape_water.png
	12 кадров по вертикали
	FPS 4.0
	размер одного кадра 640x48
	не двигается относительно камеры
	по ширине экрана (который расчетно 640х360 px)

seascape_sky.png
	* Верхняя часть тайлсета - это небо размером 640х144,
		внизу в середине пустое место для анимированного солнца.
	* Нижняя часть - 15 кадров солнца:
		размер кадра 128х48
		FPS 4.0
		направление кадров - горизонтально слева направо
		нужно вставить в пустую область неба
	Небо с солнцем не двигается относительно камеры

seascape_clouds.png
	* Для разметки см. seascape_clouds_info1.png. Здесь:
		облако большое 1 шт:
			использовать для левой и правой стороны экрана
		облако среднее 4 шт:
			по 2 шт для каждой стороны экрана
		облако маленькое 4 шт:
			по 2 шт для каждой стороны экрана
		тайл песка у воды:
			pos 0х160 size 64х64
		тайл песка в середине:
			pos 0х224 size 64х64
		тайл песка у камней:
			pos 64х224 size 64х64
			этот песок примерно в каменистом месте, т.е. низ экрана, откуда вылез краб (если понадобится)
		камни 8 шт:
			pos 256х160 size 16x16
		блики на песке 16 кадров:
			pos 128х160 size 32x32
			направление кадров - горизонтально слева направо
		синие области на разметке пустые. Можно использовать на что-то еще
	* Некоторые облака можно дублировать, делать flip и т.д.
	* Облака надо располагать по слоям примерно как в видео мокапе, где на ближнем слое - большие, на дальнем - маленькие. Позиции облаков как в мокапе см. в файле seascape_clouds_info2.png
	* Песок располагается на тайлмэпе с размером тайла 64х64
	* Камни располагаются на тайлмэпе с размером тайла 16х16. Разброс камней случайный. Плотность разброса примерно как на мокапе
	* Блики на песке анимируются ТОЛЬКО при перемещении камеры, потому что это как бы мини бугры на песке! Само положение спрайта не меняется! Смена кадра происходит примерно каждые 4 px. При перемещении камеры вправо - кадры анимируются вперед, если влево - назад.

seascape_waves.png
	* Для разметки см. seascape_waves_info.png. Здесь:
		- 4 вида берега с анимацией - по 8 кадров на каждый
		- это должны быть 4 анимированных тайла, располагаемые на тайлмэпе горизонтально друг за другом
		- FPS 4.0
		- тайлмэп должен (вероятно) генерироваться, чтобы создать длинную полосу
		- тайлмэп перемещается вместе с камерой
		- виды берегов случайны, но желательно не располагать рядом друг с другом одинаковые, чтобы тайловость не бросалась в глаза.
		- размер кадра/тайла 128x64
		- направление кадров - вертикально сверху вниз
